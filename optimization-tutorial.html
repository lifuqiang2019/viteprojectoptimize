第二阶段：体积优化 (Bundle Size Optimization)

1. 实施代码分割 (ManualChunks)

Step 1 修改 vite.config.js，配置 manualChunks 将大型第三方库拆分。

vite.config.js 
build: {
  rollupOptions: {
    output: {
      manualChunks(id) {
        if (id.includes('node_modules')) {
          // 将重型库单独拆包，只有访问对应路由时才加载
          if (id.includes('echarts')) return 'echarts';
          if (id.includes('three')) return 'three';
          if (id.includes('monaco-editor')) return 'monaco';
          if (id.includes('vue-pdf-embed')) return 'pdf';
          
          // 其他依赖合并为 vendor，避免过碎的小文件及循环依赖问题
          return 'vendor';
        }
      }
    }
  }
}

2. 开启 Tree Shaking (按需引入)

Step 2 使用 unplugin-vue-components 和 unplugin-auto-import 实现 Element Plus 的按需引入，移除未使用的组件代码。

vite.config.js 
import AutoImport from 'unplugin-auto-import/vite'
import Components from 'unplugin-vue-components/vite'
import { ElementPlusResolver } from 'unplugin-vue-components/resolvers'

plugins: [
  AutoImport({
    resolvers: [ElementPlusResolver()],
  }),
  Components({
    resolvers: [ElementPlusResolver()],
  }),
]

效果： Vendor 包体积从 ~1.2MB 进一步降低到 ~756kB。

4. 启用 PWA 离线缓存

Step 4 引入 vite-plugin-pwa，实现资源缓存，二次访问即时加载。

VitePWA({ 
  registerType: 'autoUpdate',
  workbox: {
    globPatterns: ['**/*.{js,css,html,ico,png,svg}'], // 缓存所有静态资源
    maximumFileSizeToCacheInBytes: 10 * 1024 * 1024 // 允许缓存大文件 (如 monaco)
  }
})

第四阶段：传输层优化 (Network Optimization)

5. Gzip 压缩与生产环境模拟

Step 5 这是让 FCP 达标的关键一步。我们在构建时生成 .gz 文件，并配置服务端支持 Gzip 传输。

vite.config.js 
import viteCompression from 'vite-plugin-compression'

plugins: [
  viteCompression({
    algorithm: 'gzip',
    threshold: 10240 // 只压缩 >10kb 的文件
  })
]

New Preview Server 配置： 为了在本地真实模拟生产环境的 Gzip 效果，我们创建了自定义 Server。

server.js 
import express from 'express';
import compression from 'compression';

const app = express();
// 开启 Gzip 中间件
app.use(compression());
// 托管 dist 目录
app.use(express.static('dist'));

app.listen(4173);

修改 package.json: "preview": "node server.js"
