🚀 Vite 项目性能优化实战教程

本教程将基于项目的 main 分支（未优化状态），一步步演示如何通过代码分割（Code Splitting）、资源预加载（Prefetch）、PWA 和 Gzip 压缩将项目优化至 optimization 分支的状态。

第一步：现状分析 (Baseline)

在优化开始前，我们先对 main 分支进行打包分析。该分支采用默认配置，所有依赖容易被打包进巨大的 Vendor Chunk 中。

📦 优化前打包数据

dist/assets/index-DYI8jQaA.js        996.79 kB
dist/assets/Editor-DBvM-qil.js     3,728.53 kB  (巨大!)
dist/assets/Pdf-DIlTuxTt.js        2,453.35 kB

问题：首页加载时需要下载巨大的 JS 文件，包含了可能暂时用不到的编辑器和图表库代码，导致白屏时间长。

第二步：实施代码分割 (ManualChunks)

Step 1 修改 vite.config.js，配置 manualChunks 将大型第三方库拆分。

🔧 代码变更

// vite.config.js
build: {
  rollupOptions: {
    output: {
      manualChunks(id) {
        if (id.includes('node_modules')) {
          // 将重型库单独拆包
          if (id.includes('echarts')) return 'echarts';
          if (id.includes('three')) return 'three';
          if (id.includes('monaco-editor')) return 'monaco';
          if (id.includes('vue-pdf-embed')) return 'pdf';
          
          // 其他依赖合并为 vendor，避免过碎的小文件
          return 'vendor';
        }
      }
    }
  }
}

效果： 实现了将 3.7MB 的编辑器代码和 Echarts 代码从主包中剥离，只有访问相关页面时才会加载。

第三步：开启资源预加载 (Prefetch)

Step 2 编写自定义插件 prefetchPlugin，在空闲时预加载拆分出去的 Chunk。

🔧 代码变更

// vite.config.js
function prefetchPlugin() {
  return {
    name: 'vite-plugin-prefetch',
    transformIndexHtml(html, ctx) {
      if (!ctx.bundle) return html;
      // 自动生成 <link rel="prefetch"> 标签
      const chunks = Object.values(ctx.bundle)
        .filter(chunk => chunk.fileName.endsWith('.js') || chunk.fileName.endsWith('.css'))
        .filter(chunk => !chunk.isEntry);
        
      const prefetchTags = chunks
        .map(chunk => `<link rel="prefetch" href="/${chunk.fileName}" />`)
        .join('\n');
        
      return html.replace('</head>', `${prefetchTags}\n</head>`);
    }
  }
}

效果： 用户在浏览首页时，浏览器会在后台默默下载编辑器和图表库的代码，点击路由切换时实现"秒开"。

第四步：启用 PWA 与 Gzip 压缩

Step 3 引入 vite-plugin-pwa 和 vite-plugin-compression。

🔧 代码变更

// vite.config.js
plugins: [
  // ...
  viteCompression({
    algorithm: 'gzip',
    threshold: 10240 // >10kb 进行压缩
  }),
  VitePWA({
    registerType: 'autoUpdate',
    workbox: {
      globPatterns: ['**/*.{js,css,html,ico,png,svg}']
    }
  })
]

效果：

  • Gzip: 资源体积减少 60%-70% (如 monaco 从 4MB -> 1MB)。
  • PWA: 支持离线访问，首屏资源直接从 Cache Storage 读取，极大提升二次加载速度。

总结

通过本次优化，我们成功解决了大型应用的首屏加载性能瓶颈。 Code Splitting 降低了首屏载荷，Prefetch 保证了后续路由的流畅性，而 PWA 和 Gzip 则提供了极致的缓存和传输效率。 现在，项目已经具备了生产级的高性能表现。
